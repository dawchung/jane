# 精神科病人購物管理原型

這是一個可直接用瀏覽器開啟的單機版原型，目的是取代手寫購物登記與人工彙整流程。

## 已完成功能

- 建立本次購物日期、單人購物上限與備註
- 新增病人資料：床號、姓名、目前零用金
- 依價目表加入商品，立即計算病人本次支出
- 顯示病人零用金餘額與是否超過購物上限
- 自動彙整今日採買總表，列出每個品項要買幾件、是哪些床號購買
- 產出病人分發清單，買回後可直接依床號放入物品櫃
- 使用 localStorage 保留目前輸入資料
- 可直接列印總表

## v2 更新（本次）

- 修正「病人清單偶發無法選取」：當舊資料中的目前病人 ID 失效時，會自動切回有效病人
- 商品與購物車改為事件委派，避免清單重繪後點擊失效
- 新增自訂品項輸入（名稱 + 單價），可直接加入目前病人購物車
- 新增採買追蹤勾選：每個彙整品項可標記「已採買完成」
- 新增分發追蹤勾選：每位病人可標記「已放入病人物品櫃」
- 新增 CSV 匯出，方便護理站後續存檔或另行統計

## v2.1 更新（選單與編修）

- 商品價目表改為有層次選單：先選主分類（飲料、糖果餅乾、冰品、其他），再選次分類（價格區間）
- 透過主分類 + 次分類可縮短可見品項長度，降低捲動量
- 病人選擇後可直接修改病人資料（床號、姓名、零用金）
- 已選購品項可直接輸入數量，立即更新金額與彙整報表

## 為什麼不同手機看不到同一份資料

- 若使用 `local` 模式，資料只存在各自手機瀏覽器的 localStorage，所以不會跨裝置同步
- 若要讓不同手機看到同一份清單，必須改用共用雲端資料庫

## 已加入的共享同步結構

- 頁面上方會顯示同步模式：本機 或 雲端共享
- `config.js` 可設定同步提供者
- 目前已預留 Supabase 同步模式
- `supabase-setup.sql` 提供建立共享資料表的 SQL

## 啟用 Supabase 共享同步

1. 在 Supabase 建立一個專案
2. 到 SQL Editor 執行 [supabase-setup.sql](supabase-setup.sql)
3. 到 Project Settings 取得 `Project URL` 與 `publishable key`（或 legacy `anon key`）
4. 編輯 [config.js](config.js)，改成：

```js
window.APP_CONFIG = {
	sync: {
		provider: "supabase",
		supabaseUrl: "你的 Project URL",
		supabaseAnonKey: "你的 publishable key",
		wardId: "psych-ward-a",
		pollIntervalMs: 5000
	}
};
```

5. 重新部署 GitHub Pages

## 注意

- 目前是共享病房資料的原型，採用「最後寫入者覆蓋」
- 若要正式保存病人姓名等敏感資料，下一版建議再加登入與權限控管

## 使用方式

1. 用瀏覽器開啟 `index.html`
2. 設定今日購物日期與單人上限
3. 新增病人與其零用金
4. 選擇病人後點選商品，即可累積購物車
5. 到頁面下方檢視彙整總表並列印
